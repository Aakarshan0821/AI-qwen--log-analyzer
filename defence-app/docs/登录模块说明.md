# Defence App 登录模块说明

## 1. 文档目的

本文档用于说明 `Defence App` 登录模块的设计、配置方式、页面交互、接口行为、常见问题与排查建议。

适用对象：

- 本地开发人员
- 运维部署人员
- 需要修改登录页面或认证逻辑的维护人员

## 2. 功能概览

当前登录模块提供以下能力：

1. 访问首页前先进行身份认证
2. 通过 `.env` 配置登录账号与密码
3. 登录后写入 `HttpOnly` 会话 Cookie
4. 未登录访问受保护接口时返回 `401`
5. 提供手动退出登录能力
6. 登录页左侧带角色动画效果（鼠标跟随、眨眼、输入联动）

## 3. 涉及文件

```text
app/
├─ auth.py                     # 会话 token 生成与校验
├─ api/routes.py               # 登录、退出、鉴权拦截
├─ config.py                   # 登录相关环境变量读取
├─ templates/login.html        # 登录页模板
├─ static/login.css            # 登录页样式
└─ static/login.js             # 登录页动画与交互
```

## 4. 环境变量配置

登录模块使用以下环境变量（见 `.env.example`）：

- `AUTH_ENABLED`
  - 是否启用登录认证
  - `true`：启用（默认）
  - `false`：关闭（开发联调可用）

- `LOGIN_EMAIL`
  - 登录邮箱（账号）
  - 示例：`admin@example.com`

- `LOGIN_PASSWORD`
  - 登录密码
  - 示例：`ChangeMe123!`
  - 建议上线后立即替换为高强度密码

- `SESSION_SECRET`
  - 会话签名密钥
  - 用于服务端对 Cookie token 做 HMAC 校验
  - 生产环境必须替换为高强度随机字符串

- `SESSION_MAX_AGE_SECONDS`
  - 默认会话有效时长（秒）
  - 示例：`86400`（24 小时）
  - 勾选“记住 30 天”时，时长会按 30 倍扩展

## 5. 访问与路由行为

### 5.1 页面路由

- `GET /login`
  - 返回登录页面
  - 已登录时自动重定向到 `/`

- `POST /login`
  - 提交邮箱和密码
  - 校验成功后写入会话 Cookie，并重定向到 `/`
  - 校验失败返回登录页并显示错误提示

- `POST /logout`
  - 删除会话 Cookie
  - 重定向到 `/login`

- `GET /`
  - 未登录时重定向到 `/login`
  - 已登录时返回业务首页

### 5.2 API 鉴权行为

所有业务 API 都受登录态保护（例如 `/api/search`、`/api/detect/latest` 等）：

- 已登录：正常返回
- 未登录或会话过期：返回 `401`，提示“未登录或登录已过期，请重新登录”

## 6. 登录页面交互说明

登录页分为左右两栏：

- 左栏：角色动画区
- 右栏：登录表单区

左侧动画已实现以下交互：

1. 眼球跟随鼠标移动
2. 紫色与黑色角色随机眨眼
3. 聚焦输入框后角色短暂“互看”
4. 输入密码后角色形变与视线变化
5. 显示密码时紫色角色“偷看”动画

右侧表单支持：

- 邮箱输入
- 密码输入
- 密码显示/隐藏
- 记住 30 天
- 登录失败错误提示

## 7. 会话与安全说明

### 7.1 会话实现方式

当前会话采用签名 token + Cookie 方式：

- token 内容：`email` + `exp`（过期时间）
- 签名算法：`HMAC-SHA256`
- Cookie 属性：
  - `HttpOnly=true`
  - `SameSite=lax`
  - `Path=/`
  - `Secure` 在 HTTPS 下自动开启

### 7.2 安全建议

上线环境建议至少执行：

1. 使用随机高强度 `SESSION_SECRET`
2. 修改默认账号密码
3. 使用 HTTPS 反向代理
4. 定期轮换 `LOGIN_PASSWORD`
5. 不要把 `.env` 提交到代码仓库
6. 如需多用户，建议改造为数据库用户体系

## 8. 本地配置与启动示例

### 8.1 Windows PowerShell

```powershell
cd "E:\信抗技术项目\defence-app"
Copy-Item .env.example .env
```

编辑 `.env`：

```env
AUTH_ENABLED=true
LOGIN_EMAIL=admin@example.com
LOGIN_PASSWORD=YourStrongPassword!
SESSION_SECRET=replace-with-a-long-random-secret
SESSION_MAX_AGE_SECONDS=86400
```

启动服务：

```powershell
uvicorn app.main:app --host 0.0.0.0 --port 7860
```

访问：

- [http://127.0.0.1:7860/login](http://127.0.0.1:7860/login)

## 9. 常见问题排查

### 9.1 登录成功后仍跳回登录页

排查项：

- `SESSION_SECRET` 是否频繁变化（重启后变化会导致旧会话失效）
- 浏览器是否禁用了 Cookie
- 代理层是否错误处理了 Cookie 头

### 9.2 所有 API 都返回 401

排查项：

- 是否已经登录
- 登录后 Cookie 是否成功写入
- `AUTH_ENABLED` 是否设为 `true`

### 9.3 忘记密码如何处理

当前版本没有“找回密码”后端功能，建议：

1. 直接修改 `.env` 中的 `LOGIN_PASSWORD`
2. 重启应用使配置生效

### 9.4 想临时关闭登录认证

在 `.env` 中配置：

```env
AUTH_ENABLED=false
```

然后重启应用。

## 10. 后续扩展建议

如果你后续需要更完整认证能力，可按以下方向扩展：

1. 支持多用户与角色权限（RBAC）
2. 密码哈希存储（如 `bcrypt`）
3. 登录失败次数限制与锁定
4. 图形验证码或二次验证
5. 独立用户管理页面与审计日志
